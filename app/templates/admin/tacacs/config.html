Gaia recipe
  1. Authentication Servers → TACACS+: add this host, port 49, secret above. Then Authentication Access Order → turn TACACS+ On.
  2. Gaia maps a non-local TACACS+ user to a role named TACP-<priv-lvl>. Create both (clone adminRole) so admin works in every access method:
    • TACP-15 — when Gaia authorizes the shell (we send priv-lvl=15 for admins).
    • TACP-0 — Gaia's default role when it doesn't request authorization (some WebUI logins).
  3. Members of admins → priv-lvl=15; everyone else → 0.
  4. Log in (WebUI or SSH) with the directory password — ASCII & PAP both work; watch the live log.

Gaia usernames can't contain dots — use khalid, not jane.doe. Admin groups: admins, superadmins, superusers, administrators (override with GAIA_ADMIN_GROUPS).
